1. INTRODUCTION
===============

PRÉSENTATION DU PROJET
----------------------
Stampee Auction House est une plateforme d'enchères philatéliques en ligne 
développée dans le cadre du cours Web Dynamique. Le système permet aux 
utilisateurs de consulter, créer et participer à des enchères de timbres.

FONCTIONNALITÉS IMPLÉMENTÉES
-----------------------------
✓ Gestion des utilisateurs : Inscription, connexion, profils
✓ Système d'enchères : Création, consultation, participation
✓ Recherche avancée : Filtres multiples et recherche textuelle
✓ Système de favoris : Marquage et gestion des enchères préférées
✓ Archives : Consultation des enchères terminées
✓ Commentaires : Système d'évaluation et commentaires
✓ Interface responsive : Compatible mobile et desktop
✓ Gestion des images : Upload et affichage des timbres

PUBLIC CIBLE
------------
• Collectionneurs : Consultation et participation aux enchères
• Vendeurs : Création et gestion d'enchères
• Administrateurs : Gestion du système et modération

12. FAQ ET DÉPANNAGE
======================

QUESTIONS FRÉQUEMMENT POSÉES
----------------------------

Q: Comment fonctionne le système d'enchères ?
R: Chaque utilisateur peut placer une offre supérieure à la précédente. 
   L'enchère se termine à la date spécifiée, et le plus offrant remporte 
   le timbre.

Q: Puis-je annuler une offre ?
R: Non, une fois placée, une offre est ferme et ne peut pas être retirée. 
   Réfléchissez bien avant d'enchérir.

Q: Comment sont authentifiés les timbres ?
R: Les timbres sont examinés par notre équipe d'experts philatéliques 
   avant publication.

Q: Quels sont les frais associés ?
R: Consultez notre page des tarifs pour les détails sur la structure 
   des frais.

PROBLÈMES COURANTS
------------------

Mot de passe oublié
1. Cliquez sur "Mot de passe oublié" sur la page de connexion
2. Entrez votre adresse email
3. Suivez le lien de réinitialisation reçu

Problèmes de connexion
1. Vérifiez que votre navigateur est à jour
2. Effacez le cache et les cookies
3. Vérifiez votre connexion internet

Enchères qui ne se chargent pas
1. Actualisez la page (F5)
2. Vérifiez que JavaScript est activé
3. Désactivez temporairement les extensions

15. SCRIPT DE PEUPLEMENT DE BASE DE DONNÉES LOCALE
==================================================

SCRIPT LOCAL : populate_database_local.txt
-----------------------------------------
Le projet Stampee utilise un script de peuplement de base de données locale 
pour créer des données de test et de démonstration.

INFORMATIONS TECHNIQUES
-----------------------
• **Nom du projet** : projetweb2
• **Environnement** : Base de données locale (stampee_db)
• **Fichier** : documentation/populate_database_local.txt
• **Compatible** : MySQL/MariaDB local

CONTENU DU SCRIPT
------------------
Le script crée et peuple les tables suivantes avec des données de test :
• **Utilisateurs** : Comptes de test (testuser, testuser2)
• **Timbres** : 20 timbres philatéliques avec descriptions détaillées
• **Images** : Utilise TOUS les fichiers images disponibles dans public/img/
• **Enchères** : 20 enchères actives avec dates et prix
• **Offres** : Offres de test pour simuler l'activité
• **Favoris** : Favoris de test pour démontrer la fonctionnalité

IMAGES UTILISÉES
----------------
Le script utilise une grande variété d'images pour maximiser la diversité :
• **Images de timbres** : stamp1.jpg à stamp12.jpg
• **Images de portraits** : portraitred.png, portraitgreen.jpg, etc.
• **Images historiques** : oldyellow.jpg, oldportraitbrown.jpg, etc.
• **Images artistiques** : beautyblue.jpg, tokyoblue.png, rarered.jpg
• **Images naturelles** : floweryellow.jpg, boatred.jpg, boatblue.jpg

⚠️ EXCLUSIONS : Le script exclut automatiquement :
   - background.jpg (image de fond)
   - Fichiers de paiement (payment*.png, payment*.svg, payment*.webp)
   - Fichiers de logo (redcrownlogo.png, etc.)

UTILISATION
-----------
1. **Prérequis** : Base de données locale créée avec query.sql
2. **Exécution** : Importer le script via phpMyAdmin ou ligne de commande
3. **Résultat** : Base de données complète avec données de test
4. **Test** : Vérifier que toutes les images s'affichent correctement

STRUCTURE DES DONNÉES
---------------------
• **20 enchères actives** avec images variées
• **3 images par timbre** (1 principale + 2 supplémentaires)
• **Données réalistes** : dates historiques, pays d'origine, conditions
• **Prix variés** : De 50$ à 5000$ pour tester différents scénarios

NOTES IMPORTANTES
-----------------
• Le script est conçu pour l'environnement local uniquement
• Toutes les images doivent être présentes dans public/img/
• Les chemins d'images utilisent le format 'public/img/fichier.jpg'
• Compatible avec la structure de base de données locale
